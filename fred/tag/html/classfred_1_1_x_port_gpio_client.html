Detailed Description

Implements the GPIO interface of the Lantronix XPort.

For better code readability the names of the getter and setter function differ from the names in the user guild. For instance 'get directions' does not say if 1 means input or output. It has been changed to getOutputPinsWord which makes clear that 1 means output (and hopefully implies that 0 means input and both directions are described by this word). Here is a list of the changes:

  • get functions getGpioPinMask
  • get directions getOutputPinsWord
  • get active levels getActiveLowWord
  • get current states getPinsActiveWord
  • set directions setOutputPins
  • set active levels setActiveLowWord
  • set current states setPins
