◆ readAutoPwrAlarm()

unsigned char fred::FuseRelayBoardControl::readAutoPwrAlarm ( )

Provides information whether the last boot autopwr had an error.

An autoPwr is successful (no error) if the board has not been re-booted within 5 seconds after the power has been turned on. This is enough time for the overcurrent detection to kick in. If the Fred was turned off withing these 5 seconds, the flag will report an error the next time the Fred is booted.

Return values:

  • 0 -> No error
  • 1 -> Autopwr error
  • 2 -> Five second timer still running

You likely will never see return value 2 because it usually is not possible to log into the XPort within 5 seconds after the Fred has been powered up.

void fred::FuseRelayBoardControl::setAlarmMasks ( unsigned char  overcurrentAlarmMask,
unsigned char  pin0_AlarmMask,
unsigned char  pin1_AlarmMask 
)

Configure which channels should turn off in case of an alarm either due to overcurrent or a signal on the hardware alarm pins 0 and 1.

The alarm masks are 8 bit hex values configuring whether the alarm is active for the specific channels [0..7]

  • bit n = 0 : alarm on channel n is off
  • bit n = 1 : alarm on channel n is active
    Parameters
    overcurrentAlarmMaskConfigures which channels are turned of if the channel specific current is exceeded (see setCurLimits).
    pin0AlarmMask/pin1AlarmMaskConfigures which channels are turned off if an alarm is signaled on alarm pins 0 and 1, respectively.

◆ setPower()

void fred::FuseRelayBoardControl::setPower ( unsigned char  powerStates,
unsigned char  channelMask = 0xFF 
)

Set the power state (on/off) according to the powerStates word.

The bits [0..7] represent power states of the eight channels. The new state is only assigned to the channels specified by the channel mask.

Example:

  • powerStates = 0x13
    channels 0, 1 and 4 have power state on, all other channels have state off
  • channelMask = 0x07
    Only the power states for channels 0, 1 and 2 are used. Channels 0 and 1 will be turned on, channel 2 is turned off. All other channels stay unchanged.
