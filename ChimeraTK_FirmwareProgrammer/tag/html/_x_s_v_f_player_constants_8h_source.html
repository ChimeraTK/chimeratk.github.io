88/*****************************************************************************
89 * Struct: SXsvfInfo
90 * Description: This structure contains all of the data used during the
91 * execution of the XSVF. Some data is persistent, predefined
92 * information (e.g. lRunTestTime). The bulk of this struct's
93 * size is due to the lenVal structs (defined in lenval.h)
94 * which contain buffers for the active shift data. The MAX_LEN
95 * #define in lenval.h defines the size of these buffers.
96 * These buffers must be large enough to store the longest
97 * shift data in your XSVF file. For example:
98 * MAX_LEN >= ( longest_shift_data_in_bits / 8 )
99 * Because the lenVal struct dominates the space usage of this
100 * struct, the rough size of this struct is:
101 * sizeof( SXsvfInfo ) ~= MAX_LEN * 7 (number of lenVals)
102 * xsvfInitialize() contains initialization code for the data
103 * in this struct.
104 * xsvfCleanup() contains cleanup code for the data in this
105 * struct.
106 *****************************************************************************/
107
