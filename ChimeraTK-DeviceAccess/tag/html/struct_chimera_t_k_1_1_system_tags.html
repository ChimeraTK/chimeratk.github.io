Detailed Description

A list of pre-defined tags that can be used to mark registers.

Other components (e.g. ApplicationCore) in the framework can make use of these tags to change behavior towards those registers depending on presence of those tags.

The structure of a system tag consists of three parts separated by "_": _ChimeraTK_ModuleName_tagName.

  • The prefix "_ChimeraTK" classifies this as a system tag
  • It is followed by a module name that can be used to group tags together
  • The name of the tag

Used to flag a register as "reverse recovery".

ApplicationCore can use this information to not write into this register on device recovery, but pull the latest value from it instead. Can also be used on write-only registers.
