This module is used to read the watchdog log file, which includes messages from the watchdog process and all other processes controlled by the watchdog.

This allows to show all watchdog related messages via the tail (status/logTailExternal). The LogFileModule is used differently for all the processes, since in case of the processes the massages produced by the started processes are read and shown in the corresponding tail (status/logTailExternal). Thus in that case the postfix 'External' makes more sense. Here the postfix 'External' can be a bit misleading, because the tail (status/logTailExternal) includes only messages produced by the watchdog itself.

Remarks
It would be nice to connect the message output variable of each process to its LoggingModule AND the watchdog LoggingModule. But this is not possible, since it is not possible to connect multiple outputs to a single push input variable.
