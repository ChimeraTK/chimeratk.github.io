7// All bits from the ADD_VARIABLE are only in this file. Their correctness is
8// tested with a unit test (second, independent implementation of the same
9// information). I intentionally decided to directly write the numbers in the
10// code and not define constants for each start and stop bit. These constants
11// whould only tempt users to use them in their code and start bit-fiddeling
12// themselves. In addition, having a header with so many constants would not
13// make it more "storing them all in one place" than having the numbers here,
14// within their context and two numbers, which belong together, per line.
