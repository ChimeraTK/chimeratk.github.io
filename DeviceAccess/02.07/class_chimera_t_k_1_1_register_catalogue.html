◆ catalogue

std::vector<boost::shared_ptr<RegisterInfo> > ChimeraTK::RegisterCatalogue::catalogue
protected

Vector of RegisterInfo entries.

A more efficient way to store this information would be a tree-like structure, but this can be optimised later without an interface change.

Note, to be compatible with the original RegisterInfoMap used for the MemoryAddressedBackends the data storage must retain the order of elements as they are added and allow duplicate entries. Both features are not critical for a proper functioning, but dropping them is a small interface change.
