Detailed Description

The AsyncDomain is the thread-safe entry point for each distribution tree.

Distributing data to accessors, sending exceptions and subscription of new accessors will all happen from different threads. This class implements a central mutex such that only one operation on the distribution three is executed at the same time.

This base class is providing the mutex and the _isActive flag, which is needed throughout the distribution tree. It also has a virtual setException() function to a allow sending exception from code that does not know about the distributor type.

All other functions depend on a backend-specific data type and the according distributor. Hence the AsyncDomainImpl class is templated on both.
