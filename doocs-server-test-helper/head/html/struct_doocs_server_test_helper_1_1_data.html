std::unique_lock<std::mutex> DoocsServerTestHelper::Data::update_lock {update_mutex}

unique locks for controlling update() and interrupt_usr1() from the test thread.

We keep this here in this static Data struct, because the use of these locks is "inverted" compared to normal lock use, i.e. we normally keep the mutex locked and only unlock it temporarily in runUpdate()/runSigusr1().
