11 /* the lenVal structure is a byte oriented type used to store an */
12 /* arbitrary length binary value. As an example, the hex value */
13 /* 0x0e3d is represented as a lenVal with len=2 (since 2 bytes */
14 /* and val[0]=0e and val[1]=3d. val[2-MAX_LEN] are undefined */
15 
16 /* maximum length (in bytes) of value to read in */
17 /* this needs to be at least 4, and longer than the */
18 /* length of the longest SDR instruction. If there is, */
19 /* only 1 device in the chain, MAX_LEN must be at least */
20 /* ceil(27/8) == 4. For 6 devices in a chain, MAX_LEN */
21 /* must be 5, for 14 devices MAX_LEN must be 6, for 20 */
22 /* devices MAX_LEN must be 7, etc.. */
23 /* You can safely set MAX_LEN to a smaller number if you*/
24 /* know how many devices will be in your chain. */
25 /* #define MAX_LEN (Actual #define is below this comment block)
26  This #define defines the maximum length (in bytes) of predefined
27  buffers in which the XSVF player stores the current shift data.
28  This length must be greater than the longest shift length (in bytes)
29  in the XSVF files that will be processed. 7000 is a very conservative
30  number. The buffers are stored on the stack and if you have limited
31  stack space, you may decrease the MAX_LEN value.
32 
33  How to find the "shift length" in bits?
34  Look at the ASCII version of the XSVF (generated with the -a option
35  for the SVF2XSVF translator) and search for the XSDRSIZE command
36  with the biggest parameter. XSDRSIZE is equivalent to the SVF's
37  SDR length plus the lengths of applicable HDR and TDR commands.
38  Remember that the MAX_LEN is defined in bytes. Therefore, the
39  minimum MAX_LEN = ceil( max( XSDRSIZE ) / 8 );
40 
41  The following MAX_LEN values have been tested and provide relatively
42  good margin for the corresponding devices:
43 
44  DEVICE MAX_LEN Resulting Shift Length Max (in bits)
45  --------- ------- ----------------------------------------------
46  XC9500/XL/XV 32 256
47 
48  CoolRunner/II 256 2048 - actual max 1 device = 1035 bits
49 
50  FPGA 128 1024 - svf2xsvf -rlen 1024
51 
52  XC18V00/XCF00
53  1100 8800 - no blank check performed (default)
54  - actual max 1 device = 8192 bits verify
55  - max 1 device = 4096 bits program-only
56 
57  XC18V00/XCF00 when using the optional Blank Check operation
58  2500 20000 - required for blank check
59  - blank check max 1 device = 16384 bits
60 */
61 #define MAX_LEN 7000
