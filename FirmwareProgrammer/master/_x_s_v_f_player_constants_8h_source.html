94 /*****************************************************************************
95  * Struct: SXsvfInfo
96  * Description: This structure contains all of the data used during the
97  * execution of the XSVF. Some data is persistent, predefined
98  * information (e.g. lRunTestTime). The bulk of this struct's
99  * size is due to the lenVal structs (defined in lenval.h)
100  * which contain buffers for the active shift data. The MAX_LEN
101  * #define in lenval.h defines the size of these buffers.
102  * These buffers must be large enough to store the longest
103  * shift data in your XSVF file. For example:
104  * MAX_LEN >= ( longest_shift_data_in_bits / 8 )
105  * Because the lenVal struct dominates the space usage of this
106  * struct, the rough size of this struct is:
107  * sizeof( SXsvfInfo ) ~= MAX_LEN * 7 (number of lenVals)
108  * xsvfInitialize() contains initialization code for the data
109  * in this struct.
110  * xsvfCleanup() contains cleanup code for the data in this
111  * struct.
112  *****************************************************************************/
