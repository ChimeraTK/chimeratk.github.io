Detailed Description

template<typename D_type, typename basicType, bool asyncOnGet = false, typename ActionType = basicType>
class D_fctExtended< D_type, basicType, asyncOnGet, ActionType >

A class that implements the component pattern design in DOOCS properties.

Should work well with D_float, D_int, D_string and other similar types. Example declarations: D_fctExtended < D_float, float> floatProperty; D_fctExtended <D_string, const char *> stringProperty.

Template parameters:

  • D_type The doocs type, for instance D_int
  • basicType The basic data type of the D_type, for instance int32_t for D_int
  • asyncOnGet FIXME What does this do?
  • ActionType The type of the underlying action/accessor. Usually this is the basicType of the D_type, but it might be different in special cases. One example is a unsigned 32 bit integer coming from the hardware, and Doocs does not have an unsigned 32 bit integer. If the full dynamic range is needed, the internal fixed point converter would throw an overflow exception if the signed representation would be negative (e.g. 0xffffffff = -1). In this case set the ActionType to uint32_t, together with D_type=D_int and basic_type=int32_t, and there will be an implicit C++ conversion.
